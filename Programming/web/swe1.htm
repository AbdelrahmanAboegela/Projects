Order Processing System Requirements

Requirement Type Description
Functional
  • Receive and record online orders from customers.
  • Capture items and expected delivery deadlines from customers.
  • Prioritize orders based on stock availability, transport capacity, delivery deadlines, and time.
  • Calculate the price of the order.
  • Determine the delivery date.
  • Provide customers with the option to confirm or reject orders.
Non-Functional
  • Response Time: The system should provide a response time of less than five seconds.
  • Usability: The system should be easy-to-use for employees.
  • Availability: The system must operate 24/7 to process orders.
  • Security: Implement security measures for user authentication and data protection.
